# App Store 掲載素材(ストアリスティング)下書き

作成日: 2026-07-15。App Store Connect の各欄にそのまま貼れる形でまとめる。
文字数制限: 名前30字 / サブタイトル30字 / プロモーションテキスト170字 /
キーワード100字(カンマ区切り・スペース不要)/ 概要4000字。

---

## 1. アプリ名(30字以内)

「PhotoUploader」単体は一般名詞的で、検索に埋もれる・類似名と衝突するリスクが
高い。**名前に用途を含めた B案を推奨**。

| 案 | 名前 | 字数 | 備考 |
|---|---|---|---|
| A | PhotoUploader | 13 | 現状のまま。ブランド弱い |
| B(推奨) | PhotoUploader 自分のAWSに写真バックアップ | 28 | 検索語(AWS/写真/バックアップ)を名前に含められる |
| C | じぶんクラウド写真バックアップ | 15 | 和名に振り切る案。Bundle名とは別でOK |

- ストア名はアプリ本体の `CFBundleDisplayName`(ホーム画面名)と別に設定できる。
  ホーム画面は「PhotoUploader」のままでよい
- 提出前に App Store で同名アプリがないか最終確認すること

## 2. サブタイトル(30字以内)

> 写真・動画を自分のAWSへ。サブスクなし

(代案: 「アクセスキー不要で自分のS3にバックアップ」)

## 3. プロモーションテキスト(170字以内・審査なしで随時変更可)

> 写真の保存先、借りるのをやめませんか?このアプリはあなた自身のAWSに写真・動画をバックアップします。月額料金なし、かかるのはAWSの実費だけ(100GBで月75円〜)。開発者のサーバーにデータは一切送られません。設定はブラウザでのワンクリック構築、約15分で完了します。

## 4. 概要(説明文)

> ■ あなたの写真は、あなたのクラウドに
> PhotoUploader は、iPhone の写真・動画を「あなた自身の AWS アカウント」にバックアップするアプリです。開発者のサーバーは存在せず、写真もアカウント情報も第三者に預けません。サービス終了でデータが消える心配とは無縁です。
>
> ■ 月額サブスクなし。使った分だけ
> アプリは無料。かかるのは AWS のストレージ実費だけです。節約モード(S3 Glacier Instant Retrieval)なら写真 100GB で月75円ほど。標準モードでも400円ほどです。
>
> ■ AWS の知識は不要です
> アプリ内の「はじめてガイド」から、ブラウザだけで保存先を自動構築(約15分・ワンクリック構築)。他の S3 系アプリと違い、アクセスキーの発行・貼り付けは必要ありません。長期キーを端末に置かない、より安全な設計です(ログイン+署名付きURL方式)。
>
> ■ 主な機能
> ・写真・動画の一括バックアップ(枚数無制限・4並列)
> ・バックグラウンド転送:ロックしても他のアプリを使っても継続
> ・一度アップした写真は自動スキップ(二重バックアップ防止)
> ・保存済みの一覧・拡大表示・動画再生
> ・「端末に保存」でいつでも復元(機種変更にも)
> ・保存モード切り替え(標準/節約)で料金を最適化
> ・家族で1つの保存先を共有(ユーザーごとに領域を自動分離)
> ・アップロード履歴、パスワードリセット、アプリ内アカウント削除
>
> ■ セキュリティ
> ・保存先の S3 バケットは非公開+サーバーサイド暗号化
> ・通信はすべて HTTPS、認証情報は iOS の Keychain に保存
> ・広告は非パーソナライズ配信のみ(トラッキングなし)
> ・ソースコードは GitHub で全公開
>
> ■ ご利用には
> AWS アカウント(無料作成・クレジットカード登録が必要)と、初回のみ約15分のセットアップが必要です。保存量に応じた AWS 利用料はお客様の AWS アカウントに直接請求されます。
>
> サポート・お問い合わせ: ［LPのURL］

## 6. カテゴリ・レーティング・その他

- **カテゴリ**: プライマリ「写真/ビデオ」、セカンダリ「ユーティリティ」
- **年齢レーティング**: 質問票はすべて「なし」で 4+ 想定。ただし**広告あり**の
  設問に注意(NPA でも「広告を含む」は「はい」)
- **配信地域**: 日本のみでスタート(UI・ガイドが日本語のみのため)
- **価格**: 無料
- **プライバシーポリシーURL**: `https://<ユーザー名>.github.io/photouploader/privacy.html`
- **サポートURL**: LP(`https://<ユーザー名>.github.io/photouploader/`)
- **App Privacy**: `notes/app-privacy-label.md` の表を転記
- **Review Notes**: `notes/review-demo.md` §4 を使用

## 7. スクリーンショット計画

**必須**: 6.9インチ(iPhone 16 Pro Max 等、1320×2868)1セット(1〜10枚)。
iPad 対応として提出する場合は 13インチ iPad セットも必要。

**CI流用の検討結果**: 現在のCIは iPhone 16(6.1インチ・1179×2556)で撮影して
おり、**そのままではサイズ要件を満たさない**。また撮影できるのは設定画面・
ログイン画面までで、主力になるサインイン後の画面(バックアップ進捗・
ギャラリー)はバックエンド接続が必要なので撮れない。

→ **推奨手順**: デモスタック(`notes/review-demo.md` §1)を作った後、
Mac の iPhone 16 Pro Max シミュレータ(または実機)で以下を手動撮影する。
1. はじめてガイド(「AWS知識不要」の訴求)
2. バックアップ進捗画面(複数アップロード中)
3. 保存済みギャラリー(グリッド)
4. 拡大表示+「端末に保存」
5. 保存モード選択(料金の訴求)

撮影後、文言入りフレーム(見出しテキスト)を付けるかは任意。まずは素の
スクショ5枚で提出可。

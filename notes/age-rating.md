# 年齢レーティング・配信地域 回答メモ(App Store Connect転記用)

作成日: 2026-07-19。App Store Connect の「App情報」→ 年齢制限指定の質問票に
転記するだけの状態にまとめたもの。設問の文言は画面表示が正なので、
趣旨が同じ設問に読み替えて回答する。

## 前提: 2025年の制度刷新

- 年齢区分は **4+ / 9+ / 13+ / 16+ / 18+** の5段階(旧12+/17+は廃止)
- 質問票も刷新され、コンテンツテーマに加えて **アプリ内コントロール・
  機能(Capabilities)・医療/ウェルネス** の設問が追加された
- 既存アプリは2026-01-31までに回答必須だった(新規はアプリ登録時に回答)
- レーティングは回答から自動算出。ポリシー上の理由があれば手動で
  **上げる**ことだけ可能(下げられない)

## 1. 質問票の推奨回答

### コンテンツテーマ(すべて「なし」)

| 設問 | 回答 | 根拠 |
|---|---|---|
| 暴力(現実的/漫画的/流血 等) | なし | 写真バックアップツール。アプリ自身のコンテンツは UI のみ |
| 性的内容・ヌード | なし | 同上 |
| 不敬な表現・下品なユーモア | なし | 同上 |
| アルコール・タバコ・薬物 | なし | 同上 |
| ホラー・恐怖 | なし | 同上 |
| ギャンブル(シミュレーション含む) | なし | 同上 |
| 賞金コンテスト | なし | 同上 |

### 医療・ウェルネス

| 設問 | 回答 |
|---|---|
| 医療・健康関連のトピックを扱うか | いいえ |

### 機能(Capabilities)

| 設問 | 回答 | 根拠 |
|---|---|---|
| ユーザー生成コンテンツ(共有・公開) | **いいえ** | 写真は本人の S3 に保存され本人しか閲覧できない。共有・公開・他ユーザー閲覧の機能が一切ない(SNS 的 UGC に該当しない) |
| メッセージ・チャット機能 | いいえ | なし |
| 無制限のWebアクセス(アプリ内ブラウザ) | **いいえ** | アプリ内ブラウザなし。ガイドのリンクは外部の Safari で開く |
| 広告の表示 | **はい** | AdMob(サードパーティ広告)。バナー・リワード・App Open の3種。**NPA(非パーソナライズ)のみでも「広告あり」は「はい」**(store-listing.md §のメモと同じ注意) |
| 広告がパーソナライズされるか(設問があれば) | いいえ | 全リクエスト npa=1 固定。トラッキングなし・ATT なし |

### アプリ内コントロール

| 設問 | 回答 |
|---|---|
| ペアレンタルコントロール等の年齢管理機能 | いいえ(該当機能なし) |

### 想定される算出結果

コンテンツは全て「なし」だが、**サードパーティ広告あり**のため 4+ に
ならない可能性がある(9+ 〜 13+)。算出結果が何であれ**そのまま受け入れて
提出してよい**(手動で上げる必要はない。広告は非パーソナライズのみで
子ども向けに不適切な最適化はされない前提)。

> ⚠️ このアプリは「子ども向け(Kids カテゴリ)」にはしない。Kids カテゴリは
> 広告要件が別次元に厳しく(AdMob 側も TFCD 対応が必要)、本アプリの想定
> ユーザーとも合わない。

## 2. 配信地域(日本のみ)

**App Store Connect → 対象アプリ → 「価格および配信状況」→ 配信可能地域** で
**日本のみ**を選択する。

理由:

- UI・ストア素材・サポート窓口が日本語のみ
- EEA/英国を外すことで GDPR 広告同意(認定 CMP/UMP SDK)要件が実務上無関係になる
  (AdMob 側の GDPR メッセージは公開済み・2026-07-19。PROJECT_NOTES 課題対応参照)
- EU の DSA(デジタルサービス法)による「トレーダー」情報公開(住所等)の
  義務も EU 配信を外せば回避できる

将来英語対応して配信地域を広げる場合は、UMP SDK 導入(EEA 同意取得)と
DSA トレーダー申告を先に済ませること。
